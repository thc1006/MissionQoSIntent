# ADR-0001：整合專題 A（mission→QoS）與 B（機制層）為單一論文

- 狀態：Accepted
- 日期：2026-06-25
- 決策者：Hsiu-Chi Tsai
- 諮詢：投稿決策報告（修訂版 v2，2026-06-25）
- 知會：指導教授、論文委員

## Context and Problem Statement 脈絡與問題

作者手上有兩個前置專題：(A)「mission-semantics → QoS」——把任務語意轉成可計算的 QoS 參數；(B) 機制層——GroupTB、Saturation Gate、deferred-red。問題：應把兩者**合併為單一論文**，還是各自獨立發表？合併能否形成一個夠強、夠連貫的研究問題，而非「兩個半成品釘在一起」？

## Decision Drivers 決策驅動力

- 需要一個**單一、可辯護的研究問題**，而非兩個鬆散主題。
- 單人研究生在論文週期內的**可完成性**。
- 新穎性需集中火力（見 ADR-0004：機制本身已擁擠）。
- 與既有發表（衛星任務編譯器）的**連續性**。

## Considered Options 考慮過的選項

1. **整合為一**：A 提供研究問題（mission→QoS），B 提供實作機制，統一問題為「契約→可驗證跨層控制平面」。
2. **只做 A**：純粹研究 mission→QoS 映射，不碰機制。
3. **只做 B**：純粹研究 deferred-red 等機制，不碰任務語意。
4. **兩篇獨立論文**：A、B 各自成文。

## Decision Outcome 決策結果

選擇 **「選項 1：整合為一」**，因為唯有把 A 當「為什麼（研究問題）」、B 當「怎麼做（機制）」綁在一條編譯管線上，才能形成單一可辯護的貢獻軸——**「把宣告式任務/租戶 QoS 契約，經政策守門的型別化 IR，編譯成可驗證、跨層的 admission/scheduling 控制平面」**——而不是兩個各自都已被前案逼近的子題。

### Consequences 後果

- 好處：研究問題單一、連貫；新穎性可集中在「編譯器/IR/跨層一致性」這條最少被佔據的軸；與既有編譯器發表連續（同一方法學擴展）。
- 壞處：五層整合對單人偏重（見 SDD §11 R-6）；需嚴格收斂自建範圍（只做 L2 gate + L3 節流 + 編譯器，L1/L5 消費既有）。
- 中性：A 與 B 的相對篇幅須隨 E3 實證結果調整——若 deferred-red 窄縫站不住，重心進一步往 A/編譯器傾斜。

### Confirmation 確認

論文 introduction 能用一句話陳述研究問題且不需「and also」；E1–E3 實驗都服務同一問題；reviewer 不會問「這兩部分為何在同一篇」。

## Pros and Cons of the Options 各選項利弊

### 選項 1：整合為一
- 優點：單一研究問題；火力集中；與既有發表連續；跨層一致性這條腿天然需要 A+B 才成立。
- 缺點：範圍偏大；需嚴格收斂。

### 選項 2：只做 A
- 優點：範圍小、聚焦。
- 缺點：「mission→QoS 映射」缺乏系統落地與實證，偏概念；難在系統會議站住。

### 選項 3：只做 B
- 優點：範圍小、可實作。
- 缺點：機制層已擁擠（SCORPIO/Niyama/GAIE，見 ADR-0004）；缺領域動機，淪為「又一個 SLO scheduler」。

### 選項 4：兩篇獨立
- 優點：各自簡單。
- 缺點：單人難產兩篇；各自新穎性都弱；切開後反而失去「跨層一致性」這個唯一強腿。
